# UI Engineer (UI Designer) Skills

The UI Engineer (or UI Designer) focuses on the visual aspects of the product, ensuring it is aesthetically pleasing, consistent, and intuitive.

## Role Definition
Focuses on visual design, branding, layout, typography, color theory, and high-fidelity prototyping.

## Skills & Tools

### 1. Visual Design
Creating the look and feel of the product.

- **Tool: Design Systems**
  - *Description*: A collection of reusable components, guided by clear standards.
  - *Practice*: Maintain consistency across the product.
  - *Agent Prompt*: "Define the color palette and typography for the new Design System."

- **Tool: Typography & Color Theory**
  - *Description*: Choosing fonts and colors that align with the brand and improve readability.
  - *Practice*: Select accessible color contrasts and readable font pairings.
  - *Agent Prompt*: "Suggest a color scheme for a health and wellness app."

### 2. Prototyping
Creating interactive mockups.

- **Tool: High-Fidelity Prototypes**
  - *Description*: Prototypes that look and function closely to the final product.
  - *Practice*: Use tools like Figma, Sketch, or Adobe XD.
  - *Agent Prompt*: "Describe the interactions for a high-fidelity prototype of the checkout flow."

- **Tool: Micro-interactions**
  - *Description*: Subtle animations that provide feedback (e.g., button hover, loading spinner).
  - *Practice*: Enhance user delight and usability.
  - *Agent Prompt*: "Design a micro-interaction for the 'Like' button."

### 3. Layout & Responsiveness
Ensuring the design works on all devices.

- **Tool: Grid Systems**
  - *Description*: A structure for organizing content on a page.
  - *Practice*: Use 12-column grids for responsive web design.
  - *Agent Prompt*: "Design a responsive grid layout for the dashboard."

- **Tool: Responsive Design**
  - *Description*: Adapting the layout to different screen sizes.
  - *Practice*: Design for Mobile, Tablet, and Desktop breakpoints.
  - *Agent Prompt*: "Create a mobile-first layout for the landing page."

### 4. Collaboration
Working with developers and UX designers.

- **Tool: Handoff Tools**
  - *Description*: Tools to share specs with developers (Zeplin, Figma Dev Mode).
  - *Practice*: Provide redlines, assets, and CSS snippets.
  - *Agent Prompt*: "Prepare a design handoff document for the login screen."

- **Tool: Style Guides**
  - *Description*: Documentation of design standards.
  - *Practice*: Ensure developers implement the design correctly.
  - *Agent Prompt*: "Create a style guide for buttons and form inputs."
