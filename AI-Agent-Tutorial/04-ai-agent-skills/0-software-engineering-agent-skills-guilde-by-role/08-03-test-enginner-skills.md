# Test Engineer Skills

The Test Engineer is responsible for executing test cases, identifying defects, and ensuring the software functions as expected.

## Role Definition
Focuses on manual testing, automated test execution, bug reporting, and regression testing.

## Skills & Tools

### 1. Test Execution
Running tests to verify functionality.

- **Tool: Test Management Tools**
  - *Description*: Tools to manage test cases and execution (TestRail, Zephyr).
  - *Practice*: Execute test runs and record results (Pass/Fail).
  - *Agent Prompt*: "Execute the 'Checkout Process' test suite and log results."

- **Tool: Cross-Browser Testing**
  - *Description*: Testing on different browsers and devices (BrowserStack, Sauce Labs).
  - *Practice*: Ensure compatibility across Chrome, Safari, Firefox, and mobile devices.
  - *Agent Prompt*: "Test the landing page responsiveness on iPhone 14 and iPad Pro."

### 2. Defect Reporting
Communicating issues clearly.

- **Tool: Bug Tracking (Jira)**
  - *Description*: Standard tool for issue tracking.
  - *Practice*: Create detailed bug tickets with steps to reproduce, expected vs. actual results, and severity.
  - *Agent Prompt*: "Draft a bug report for a 404 error on the payment page."

- **Tool: Screen Capture & Recording**
  - *Description*: Tools to capture evidence (Snagit, Loom).
  - *Practice*: Attach screenshots or videos to bug reports.
  - *Agent Prompt*: "Describe how to capture network logs for a bug report."

### 3. Automated Testing
Writing and running scripts.

- **Tool: Selenium / Cypress**
  - *Description*: Web automation frameworks.
  - *Practice*: Write scripts to automate repetitive regression tests.
  - *Agent Prompt*: "Write a Selenium script in Python to login and verify the dashboard loads."

- **Tool: API Testing**
  - *Description*: Verifying backend endpoints (Postman, SoapUI).
  - *Practice*: Send requests and validate responses.
  - *Agent Prompt*: "Create a data-driven test in Postman for the 'User Registration' API."

### 4. specialized Testing
Testing specific aspects of the software.

- **Tool: Accessibility Testing**
  - *Description*: Ensuring usability for all (WAVE, AXE).
  - *Practice*: Check for WCAG compliance.
  - *Agent Prompt*: "Run an accessibility audit on the homepage using the WAVE tool."

- **Tool: Security Testing (Basic)**
  - *Description*: Checking for basic vulnerabilities.
  - *Practice*: Test for SQL injection or XSS using input fields.
  - *Agent Prompt*: "Test the search bar for basic XSS vulnerabilities."
