# UE Engineer (User Experience) Skills

*Note: In many organizations, UE (User Experience) and UX (User Experience) are used interchangeably. This file focuses on the deeper psychological and research aspects, distinct from pure interaction design.*

## Role Definition
Focuses on the holistic journey, emotional response, accessibility, and deep user research.

## Skills & Tools

### 1. Advanced User Research
Going beyond surface-level needs.

- **Tool: Ethnographic Studies**
  - *Description*: Observing users in their natural environment.
  - *Practice*: Understand context and environmental factors affecting usage.
  - *Agent Prompt*: "Plan an ethnographic study for a warehouse management app."

- **Tool: Diary Studies**
  - *Description*: Users log their activities and thoughts over time.
  - *Practice*: Capture longitudinal data and habits.
  - *Agent Prompt*: "Design a diary study to track user fitness habits over 2 weeks."

### 2. Cognitive Psychology
Applying psychological principles to design.

- **Tool: Cognitive Load Theory**
  - *Description*: Understanding the limits of working memory.
  - *Practice*: Simplify interfaces to reduce mental effort.
  - *Agent Prompt*: "Analyze the dashboard for cognitive load issues."

- **Tool: Mental Models**
  - *Description*: The user's internal explanation of how a system works.
  - *Practice*: Align the interface with existing user expectations.
  - *Agent Prompt*: "Identify the user's mental model for a file management system."

### 3. Accessibility (A11y)
Designing for all abilities.

- **Tool: Screen Readers**
  - *Description*: Software that reads text on the screen (NVDA, VoiceOver).
  - *Practice*: Test navigation and content structure without sight.
  - *Agent Prompt*: "Describe how to test a navigation menu with VoiceOver."

- **Tool: WCAG Guidelines**
  - *Description*: Web Content Accessibility Guidelines.
  - *Practice*: Ensure compliance with levels A, AA, AAA.
  - *Agent Prompt*: "Check the color contrast of these buttons against WCAG AA standards."

### 4. Service Design
Designing the broader service ecosystem.

- **Tool: Service Blueprints**
  - *Description*: Visualizing the relationships between different service components (people, props, processes).
  - *Practice*: Map front-stage and back-stage actions.
  - *Agent Prompt*: "Create a Service Blueprint for a hotel check-in process."
