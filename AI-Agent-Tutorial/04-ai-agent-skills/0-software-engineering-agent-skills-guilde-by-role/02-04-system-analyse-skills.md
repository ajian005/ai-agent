# System Analyst Skills

The System Analyst (SA) bridges the gap between business requirements and technical implementation, designing the system architecture and specifications.

## Role Definition
Focuses on technical feasibility, system design, data modeling, and integration requirements.

## Skills & Tools

### 1. System Design
Designing the structure and components of the system.

- **Tool: UML Diagrams**
  - *Description*: Unified Modeling Language for visualizing system design.
  - *Practice*: Create Class diagrams, Sequence diagrams, Component diagrams.
  - *Agent Prompt*: "Create a Sequence Diagram for the 'User Login' flow."

- **Tool: C4 Model**
  - *Description*: Context, Containers, Components, Code model for software architecture.
  - *Practice*: visualize the static structure of the system at different levels.
  - *Agent Prompt*: "Draw a C4 Component diagram for the 'Payment Service'."

- **Tool: Architecture Decision Records (ADR)**
  - *Description*: A document that captures an important architectural decision made along with its context and consequences.
  - *Practice*: Document "Why" a technical decision was made.
  - *Agent Prompt*: "Draft an ADR for choosing PostgreSQL over MongoDB."

### 2. Data Modeling
Designing the data structure and relationships.

- **Tool: ERD (Entity Relationship Diagram)**
  - *Description*: Visual representation of the data entities and their relationships.
  - *Practice*: Define tables, keys, and relationships (1:1, 1:N, N:M).
  - *Agent Prompt*: "Create an ERD for an E-commerce order system."

- **Tool: Data Dictionary**
  - *Description*: A collection of descriptions of the data objects or items in a data model.
  - *Practice*: Define data types, constraints, and descriptions for each field.
  - *Agent Prompt*: "Generate a Data Dictionary for the 'User' table."

### 3. API Design
Defining the interfaces for system interaction.

- **Tool: OpenAPI (Swagger) Specification**
  - *Description*: Standard for defining RESTful APIs.
  - *Practice*: Define endpoints, request/response bodies, and error codes.
  - *Agent Prompt*: "Write an OpenAPI spec for the 'Get Product Details' endpoint."

- **Tool: Interface Contract**
  - *Description*: Agreement between service provider and consumer.
  - *Practice*: Ensure compatibility and clear expectations.
  - *Agent Prompt*: "Define the interface contract for the 'Inventory Service'."

### 4. Requirement Analysis (Technical)
Analyzing technical feasibility and constraints.

- **Tool: Feasibility Study**
  - *Description*: Assessment of the practicality of a proposed project or system.
  - *Practice*: Evaluate technical, economic, and operational feasibility.
  - *Agent Prompt*: "Conduct a technical feasibility study for real-time video processing."

- **Tool: Gap Analysis**
  - *Description*: Comparison of actual performance with potential or desired performance.
  - *Practice*: Identify the gap between current system capabilities and new requirements.
  - *Agent Prompt*: "Perform a gap analysis for the legacy migration project."
