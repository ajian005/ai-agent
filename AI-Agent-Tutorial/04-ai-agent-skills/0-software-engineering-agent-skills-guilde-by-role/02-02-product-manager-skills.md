# Product Manager Skills

The Product Manager (PM) is responsible for defining the "what" and "why" of the product, bridging the gap between business, technology, and user experience.

## Role Definition
Focuses on user needs, feature prioritization, roadmap planning, and product-market fit.

## Skills & Tools

### 1. Requirement Analysis
Translating user needs into actionable requirements.

- **Tool: PRD (Product Requirements Document)**
  - *Description*: A document that describes the product you are about to build.
  - *Practice*: Define purpose, features, functionality, and behavior.
  - *Agent Prompt*: "Draft a PRD for the new 'Dark Mode' feature."

- **Tool: User Stories**
  - *Description*: Short, simple descriptions of a feature from the perspective of the person who desires the new capability.
  - *Practice*: Format as "As a [user], I want [goal] so that [benefit]".
  - *Agent Prompt*: "Write 5 user stories for the 'Shopping Cart' feature."

### 2. Prioritization
Deciding what to build next.

- **Tool: MoSCoW Method**
  - *Description*: Prioritization technique (Must have, Should have, Could have, Won't have).
  - *Practice*: Categorize features to manage scope and expectations.
  - *Agent Prompt*: "Prioritize the backlog using the MoSCoW method."

- **Tool: RICE Score**
  - *Description*: Scoring system based on Reach, Impact, Confidence, and Effort.
  - *Practice*: Quantify the value of features to rank them objectively.
  - *Agent Prompt*: "Calculate RICE scores for the top 3 proposed features."

- **Tool: Kano Model**
  - *Description*: Theory for product development and customer satisfaction (Basic, Performance, Excitement needs).
  - *Practice*: Classify features based on how they affect customer satisfaction.
  - *Agent Prompt*: "Classify these features using the Kano Model."

### 3. User Research
Understanding the user.

- **Tool: User Personas**
  - *Description*: Fictional characters created to represent the different user types.
  - *Practice*: Create detailed profiles including goals, frustrations, and behaviors.
  - *Agent Prompt*: "Create a user persona for a 'Power User' of our analytics tool."

- **Tool: Customer Journey Map**
  - *Description*: Visual representation of every experience your customers have with you.
  - *Practice*: Map out the user's path from awareness to advocacy.
  - *Agent Prompt*: "Map the customer journey for the onboarding process."

### 4. Roadmap Planning
Visualizing the product strategy over time.

- **Tool: Product Roadmap**
  - *Description*: A shared source of truth that outlines the vision, direction, priorities, and progress of a product over time.
  - *Practice*: Plan releases and communicate timelines to stakeholders.
  - *Agent Prompt*: "Create a high-level roadmap for Q3 and Q4."

- **Tool: Release Plan**
  - *Description*: A detailed plan for a specific product release.
  - *Practice*: Coordinate development, testing, and deployment activities.
  - *Agent Prompt*: "Draft a release plan for version 2.0."
