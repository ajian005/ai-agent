# UX Engineer (UX Designer) Skills

The UX Engineer (or User Experience Designer) is responsible for designing the overall experience of the product, ensuring it is usable, accessible, and desirable. They focus on the "why" and "how" of the user's interaction.

## Role Definition
Focuses on user research, information architecture, wireframing, usability testing, and accessibility.

## Skills & Tools

### 1. User Research
Understanding user needs and behaviors.

- **Tool: User Interviews**
  - *Description*: One-on-one conversations to gather qualitative data.
  - *Practice*: Ask open-ended questions to uncover pain points and motivations.
  - *Agent Prompt*: "Draft a script for user interviews regarding the new 'Dashboard' feature."

- **Tool: Surveys**
  - *Description*: Collecting quantitative data from a large group.
  - *Practice*: Design questions to measure satisfaction or feature usage.
  - *Agent Prompt*: "Create a survey to gauge user satisfaction with the onboarding process."

- **Tool: Empathy Maps**
  - *Description*: Visualizing what users say, think, do, and feel.
  - *Practice*: Develop a deeper understanding of user emotions.
  - *Agent Prompt*: "Create an empathy map for a frustrated e-commerce user."

### 2. Information Architecture (IA)
Organizing content and navigation.

- **Tool: Card Sorting**
  - *Description*: Asking users to organize topics into categories.
  - *Practice*: Improve site navigation and menu structure.
  - *Agent Prompt*: "Plan a card sorting exercise to reorganize the Help Center."

- **Tool: Sitemaps**
  - *Description*: Visual representation of the site's hierarchy.
  - *Practice*: Define the relationships between pages.
  - *Agent Prompt*: "Create a sitemap for the new mobile application."

- **Tool: User Flows**
  - *Description*: Diagramming the path a user takes to complete a task.
  - *Practice*: Identify friction points and optimize the journey.
  - *Agent Prompt*: "Map the user flow for resetting a forgotten password."

### 3. Interaction Design & Prototyping
Designing how the product behaves.

- **Tool: Wireframing**
  - *Description*: Low-fidelity sketches of the interface structure.
  - *Practice*: Focus on layout and functionality without visual distractions.
  - *Agent Prompt*: "Create wireframes for the 'Profile Settings' page."

- **Tool: Interactive Prototyping**
  - *Description*: Building clickable models to test interactions (Figma, Axure).
  - *Practice*: Validate concepts before development.
  - *Agent Prompt*: "Describe the interactions for a prototype of the search functionality."

### 4. Usability Testing & Evaluation
Validating designs with real users.

- **Tool: Usability Testing**
  - *Description*: Observing users as they attempt tasks.
  - *Practice*: Identify usability issues and areas for improvement.
  - *Agent Prompt*: "Draft a usability test plan for the new checkout process."

- **Tool: Heuristic Evaluation**
  - *Description*: Evaluating the interface against usability principles (Nielsen's Heuristics).
  - *Practice*: Expert review to find common usability problems.
  - *Agent Prompt*: "Conduct a heuristic evaluation of the landing page."

- **Tool: Accessibility Audit (WCAG)**
  - *Description*: Ensuring the product is usable by people with disabilities.
  - *Practice*: Check for keyboard navigation, screen reader compatibility, and color contrast.
  - *Agent Prompt*: "Audit the login form for WCAG 2.1 compliance."
