# Project Manager Skills

The Project Manager (PM) is responsible for the planning, execution, monitoring, control, and closure of projects. They ensure that the project is delivered on time, within scope, and within budget.

## Role Definition
Focuses on scheduling, resource management, risk management, stakeholder communication, and process adherence.

## Skills & Tools

### 1. Project Planning & Scheduling
Defining the timeline and milestones.

- **Tool: Gantt Chart**
  - *Description*: A type of bar chart that illustrates a project schedule.
  - *Practice*: Visualize tasks, dependencies, and duration.
  - *Agent Prompt*: "Create a Gantt Chart for the 'Mobile App Launch' project."

- **Tool: WBS (Work Breakdown Structure)**
  - *Description*: Breaking down the project into smaller, manageable components.
  - *Practice*: Decompose deliverables into work packages.
  - *Agent Prompt*: "Create a WBS for the 'Website Redesign' project."

- **Tool: Critical Path Method (CPM)**
  - *Description*: Identifying the longest sequence of dependent tasks.
  - *Practice*: Determine the minimum project duration and critical tasks.
  - *Agent Prompt*: "Identify the critical path for the construction phase."

### 2. Agile Management
Managing projects using iterative and incremental approaches.

- **Tool: Kanban Board**
  - *Description*: Visualizing work, limiting work-in-progress, and maximizing flow.
  - *Practice*: Manage tasks using columns (To Do, In Progress, Done).
  - *Agent Prompt*: "Set up a Kanban board structure for the marketing team."

- **Tool: Scrum Framework**
  - *Description*: Framework for developing complex products.
  - *Practice*: Facilitate Sprints, Stand-ups, Reviews, and Retrospectives.
  - *Agent Prompt*: "Draft an agenda for the Sprint Planning meeting."

- **Tool: Burndown Chart**
  - *Description*: Tracking the amount of work remaining vs. time.
  - *Practice*: Monitor progress and predict completion dates.
  - *Agent Prompt*: "Analyze the Sprint Burndown chart to predict if we will meet the goal."

### 3. Risk Management
Identifying and mitigating potential issues.

- **Tool: RAID Log**
  - *Description*: Tracking Risks, Assumptions, Issues, and Dependencies.
  - *Practice*: Regularly review and update the RAID log.
  - *Agent Prompt*: "Create a RAID log for the software migration project."

- **Tool: Risk Register**
  - *Description*: A list of identified risks, their severity, and mitigation plans.
  - *Practice*: Prioritize risks based on impact and probability.
  - *Agent Prompt*: "Draft a risk register for the cloud migration."

### 4. Stakeholder Communication
Keeping everyone informed and aligned.

- **Tool: Status Report**
  - *Description*: Regular updates on project progress.
  - *Practice*: Communicate key achievements, blockers, and next steps.
  - *Agent Prompt*: "Write a weekly status report for the executive team."

- **Tool: RACI Matrix**
  - *Description*: Clarifying roles and responsibilities (Responsible, Accountable, Consulted, Informed).
  - *Practice*: Assign roles for each task or deliverable.
  - *Agent Prompt*: "Create a RACI matrix for the product launch."
