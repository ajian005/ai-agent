# Development Manager (Engineering Manager) Skills

The Development Manager (or Engineering Manager) is responsible for leading engineering teams, managing people, and ensuring delivery excellence.

## Role Definition
Focuses on people management, hiring, career growth, team performance, and delivery execution.

## Skills & Tools

### 1. People Management
Supporting and growing the team.

- **Tool: 1:1 Meetings**
  - *Description*: Regular check-ins with direct reports.
  - *Practice*: Discuss career goals, feedback, and blockers (not just status updates).
  - *Agent Prompt*: "Draft an agenda for a career development 1:1 meeting."

- **Tool: Performance Reviews**
  - *Description*: Formal assessment of employee performance.
  - *Practice*: Provide constructive feedback and set goals using frameworks like SMART.
  - *Agent Prompt*: "Write a performance review for a Senior Engineer exceeding expectations."

- **Tool: Career Ladders**
  - *Description*: Defined paths for professional growth.
  - *Practice*: Map individual skills to role expectations.
  - *Agent Prompt*: "Create a career development plan for a Junior Engineer moving to Mid-level."

### 2. Hiring & Onboarding
Building the team.

- **Tool: Interview Rubrics**
  - *Description*: Standardized criteria for evaluating candidates.
  - *Practice*: Ensure fair and consistent hiring decisions.
  - *Agent Prompt*: "Design an interview rubric for a Backend Engineer role."

- **Tool: Onboarding Checklists**
  - *Description*: A list of tasks to complete for new hires.
  - *Practice*: Ensure new engineers are productive quickly (access, environment setup, first commit).
  - *Agent Prompt*: "Create a 30-60-90 day onboarding plan for a new Team Lead."

### 3. Delivery Management
Ensuring the team ships software effectively.

- **Tool: Velocity & Cycle Time**
  - *Description*: Metrics to measure team throughput and speed.
  - *Practice*: Analyze trends to identify process bottlenecks.
  - *Agent Prompt*: "Analyze the Cycle Time trend for the last 3 sprints."

- **Tool: Capacity Planning**
  - *Description*: Estimating the amount of work the team can handle.
  - *Practice*: Balance roadmap commitments with team availability.
  - *Agent Prompt*: "Calculate team capacity for the upcoming quarter considering holidays."

### 4. Technical Leadership
Guiding technical decisions (without necessarily writing code).

- **Tool: Architecture Review**
  - *Description*: Evaluating system design proposals.
  - *Practice*: Ensure designs align with long-term technical strategy.
  - *Agent Prompt*: "Review the architecture proposal for scalability and maintainability risks."

- **Tool: Incident Management**
  - *Description*: Leading the team during outages.
  - *Practice*: Act as Incident Commander and ensure post-mortems happen.
  - *Agent Prompt*: "Draft a process for handling P0 production incidents."
