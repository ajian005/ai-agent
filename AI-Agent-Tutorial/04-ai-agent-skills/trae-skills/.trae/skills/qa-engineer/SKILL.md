# QA Engineer (Quality Assurance)

This skill focuses on preventing defects and ensuring the software meets requirements through rigorous testing.

## Core Responsibilities

1.  **Test Planning**: Defining what to test and how.
2.  **Test Execution**: Running manual and automated tests.
3.  **Defect Tracking**: Reporting and managing bugs.
4.  **Automation**: Writing scripts to execute tests automatically.

## Key Frameworks

### 1. Testing Pyramid
-   **Unit Tests**: Testing individual components (fast, cheap).
-   **Integration Tests**: Testing interactions between components.
-   **E2E Tests**: Testing the full user journey (slow, expensive).

### 2. Types of Testing
-   **Functional**: Does it do what it's supposed to do?
-   **Non-Functional**: Performance, security, usability.
-   **Regression**: Did we break anything existing?

## Workflow

When invoked, this skill will guide you through:

1.  **Analysis**: Review requirements and acceptance criteria.
2.  **Design**: Write test cases and scenarios.
3.  **Implementation**: Develop automated test scripts (Selenium, Cypress, Pytest).
4.  **Execution**: Run tests and analyze results.
5.  **Reporting**: Log bugs and generate test reports.

## Usage Scenarios

-   **New Feature**: "Test the login page." -> QA Skill: "Verify valid/invalid credentials, password reset, and SQL injection."
-   **Regression**: "We just deployed v2.0." -> QA Skill: "Run the smoke test suite to ensure critical paths work."
-   **Automation**: "Automate this checkout flow." -> QA Skill: "Write a Playwright/Cypress script to simulate a user purchase."

## Output Format

-   **Test Plan**: Scope, strategy, and schedule.
-   **Test Cases**: Steps, expected results, and actual results.
-   **Bug Report**: Reproduction steps, severity, and screenshots.
