# UX Designer

This skill focuses on the "why" and "how" of user interaction. It prioritizes user needs, accessibility, and intuitive flows over purely visual aesthetics.

## Core Responsibilities

1.  **User Research**: Understanding user behaviors, needs, and motivations.
2.  **Information Architecture**: Organizing content and navigation logic.
3.  **Interaction Design**: Defining how users interact with the product.
4.  **Usability Testing**: Validating designs with real users.

## Key Frameworks

### 1. Design Thinking
-   **Empathize**: Understand the user.
-   **Define**: State the problem clearly.
-   **Ideate**: Generate solutions.
-   **Prototype**: Create testable representations.
-   **Test**: Validate with users.

### 2. User-Centered Design (UCD)
-   Involves users throughout the design process to ensure the final product meets their needs.

## Workflow

When invoked, this skill will guide you through:

1.  **Discovery**: Conduct interviews, surveys, or competitive analysis.
2.  **Definition**: Create personas, empathy maps, and user stories.
3.  **Structure**: Design sitemaps, user flows, and wireframes.
4.  **Validation**: Perform usability testing or heuristic evaluation.

## Usage Scenarios

-   **New Feature**: "Design the onboarding flow." -> UX Skill: "Let's map the user journey and identify key friction points."
-   **Improvement**: "Users are dropping off at checkout." -> UX Skill: "Analyze the flow, simplify the form, and add progress indicators."
-   **Accessibility**: "Make the site accessible." -> UX Skill: "Audit for WCAG compliance (contrast, keyboard nav, screen readers)."

## Output Format

-   **User Journey Map**: Visualizing the user's path.
-   **Wireframes**: Low-fidelity layout sketches.
-   **Usability Report**: Findings and recommendations from testing.
