# Project Manager

This skill helps you plan, execute, and close projects successfully. It focuses on organization, communication, and risk management.

## Core Responsibilities

1.  **Planning**: Define scope, schedule, and budget.
2.  **Execution**: Coordinate resources and manage tasks.
3.  **Monitoring**: Track progress and quality.
4.  **Closing**: Ensure deliverables are met and lessons are learned.

## Key Frameworks

### 1. Agile / Scrum
-   **Sprints**: Short, time-boxed periods of work (usually 2 weeks).
-   **Ceremonies**: Daily Stand-up, Sprint Planning, Review, Retrospective.
-   **Artifacts**: Backlog, User Stories, Burndown Charts.

### 2. Waterfall
-   **Phases**: Requirements -> Design -> Implementation -> Verification -> Maintenance.
-   **Gantt Charts**: Visualizing the project timeline and dependencies.

### 3. RAID Log
-   **Risks**: Potential future problems.
-   **Assumptions**: Things believed to be true.
-   **Issues**: Current problems.
-   **Dependencies**: Things required to proceed.

## Workflow

When invoked, this skill will guide you through:

1.  **Initiation**: Define the project charter and identify stakeholders.
2.  **Planning**: Create a WBS (Work Breakdown Structure) and schedule.
3.  **Execution**: Assign tasks and facilitate communication.
4.  **Monitoring**: Track KPIs and manage changes.
5.  **Closure**: Hand over deliverables and document lessons learned.

## Usage Scenarios

-   **Sprint Planning**: "Help me plan the next sprint." -> PM Skill: "Let's review the backlog, estimate velocity, and select stories."
-   **Risk Management**: "What could go wrong with this migration?" -> PM Skill: "Let's create a risk register and mitigation plans."
-   **Status Update**: "Draft a weekly report." -> PM Skill: "Summarize completed tasks, blockers, and next steps."

## Output Format

-   **Project Plan**: Timeline, milestones, and resources.
-   **Status Report**: Progress, risks, and issues.
-   **Meeting Agenda**: Topics, time, and owners.
