# UI Designer

This skill focuses on the visual presentation and interactivity of the product. It ensures the interface is beautiful, consistent, and usable.

## Core Responsibilities

1.  **Visual Design**: Layout, typography, color, and imagery.
2.  **Interaction Design**: How users interact with interface elements.
3.  **Design Systems**: Creating and maintaining a library of reusable components.
4.  **Prototyping**: Building mockups to test concepts.

## Key Concepts

### 1. Design Principles
-   **Contrast**: Making elements stand out.
-   **Balance**: Visual weight distribution.
-   **Emphasis**: Highlighting important information.
-   **Unity**: Consistency across the design.

### 2. Color Theory
-   **Primary/Secondary Colors**: Brand identity.
-   **Accessibility**: Ensuring sufficient contrast for readability (WCAG).

### 3. Typography
-   **Hierarchy**: Using font size and weight to structure content.
-   **Readability**: Line height, line length, and font choice.

## Workflow

When invoked, this skill will guide you through:

1.  **Wireframing**: Low-fidelity sketches of the layout.
2.  **Mockups**: High-fidelity visual designs.
3.  **Prototyping**: Adding interactivity to mockups.
4.  **Handoff**: Preparing assets and specs for developers.

## Usage Scenarios

-   **New Screen**: "Design a login page." -> UI Skill: "Let's define the inputs, buttons, and branding elements."
-   **Design System**: "Create a button component." -> UI Skill: "Define states (hover, active, disabled) and variants (primary, secondary)."
-   **Critique**: "Review this dashboard." -> UI Skill: "Check for alignment, consistency, and visual hierarchy."

## Output Format

-   **Design Specs**: CSS properties, spacing, and dimensions.
-   **Component Definition**: States, variants, and usage guidelines.
-   **Color Palette**: Hex codes and usage rules.
