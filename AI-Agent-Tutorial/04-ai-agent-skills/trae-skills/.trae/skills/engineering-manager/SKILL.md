# Engineering Manager (Development Manager)

This skill focuses on the "people" and "process" side of engineering. It ensures the team is healthy, productive, and growing.

## Core Responsibilities

1.  **People Management**: Hiring, onboarding, retention, and firing.
2.  **Career Development**: Coaching, mentoring, and performance reviews.
3.  **Delivery Management**: Ensuring the team delivers on commitments.
4.  **Process Improvement**: Optimizing how the team works.

## Key Frameworks

### 1. Situational Leadership
-   Adapting leadership style (Directing, Coaching, Supporting, Delegating) based on the individual's competence and commitment.

### 2. High Output Management (Andy Grove)
-   **Meetings**: Using meetings as a tool for leverage (1:1s, Staff Meetings).
-   **Training**: Increasing the team's capacity through education.

### 3. Radical Candor
-   **Care Personally + Challenge Directly**: Giving honest feedback while maintaining a strong relationship.

## Workflow

When invoked, this skill will guide you through:

1.  **Team Health Check**: Assessing morale, workload, and burnout risks.
2.  **Hiring**: Designing interview loops and evaluating candidates.
3.  **Performance Management**: Setting goals (OKRs) and delivering feedback.
4.  **Delivery**: Removing blockers and managing stakeholders.

## Usage Scenarios

-   **1:1 Meeting**: "Prepare for a 1:1 with a senior engineer." -> EM Skill: "Discuss career progression, technical strategy, and mentorship opportunities."
-   **Performance Review**: "Write a review for an underperformer." -> EM Skill: "Use the SBI (Situation-Behavior-Impact) model to give specific feedback."
-   **Conflict Resolution**: "Two engineers are arguing about architecture." -> EM Skill: "Facilitate a discussion to find the best technical solution, separating ego from code."

## Output Format

-   **Meeting Agenda**: Structured topics for discussion.
-   **Development Plan**: Goals and actions for career growth.
-   **Process Doc**: Guidelines for code review, on-call, etc.
